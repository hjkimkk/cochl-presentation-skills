Round 1 · Design ReviewJun 2026hjkim · cochlear.ai
Onboarding Flow RedesignJun 2026
Direction 1
Streamlined Device Management
Reorganise the Devices tab with explicit visual hierarchy between Projects, Groups, and Devices. Inline quick-actions reduce the number of taps for common tasks.

SDK and API project types are clearly differentiated through icon and badge treatments, helping users build a mental model faster.
Keywords: Hierarchy  ·  Grouping  ·  Inline Actions
Onboarding Flow RedesignJun 2026
Direction 2
Tag-Driven Discovery
Elevate tags to a primary navigation layer. The bottom sheet becomes the central discovery tool — proactively suggesting relevant tags based on device type and recent event history.

The Summary tab becomes the default home after initial setup. Tag selection is a first-class action, not a buried menu option.
Keywords: Tags  ·  Discovery  ·  Context-First
Onboarding Flow RedesignJun 2026
Direction 3
Progressive Onboarding
A guided first-run experience introducing Cochl.Sense's core concepts — devices, groups, and tags — in sequence through contextual prompts.

Each step reveals the next layer only when relevant, reducing cognitive load without hiding power features. Experienced users can skip ahead at any point.
Keywords: Onboarding  ·  Progressive Disclosure  ·  Guided Setup
