Overview

Claude Code Skills란?

Claude Code에 등록된 슬래시 커맨드(/)로 호출하는 AI 워크플로우 모듈입니다.
복잡한 프롬프트 없이, 명령어 하나로 Cochl 브랜드에 맞는 결과물을 즉시 생성합니다.
핵심 개념: 스킬은 Cochl 디자인 토큰, 레이아웃 문법, QA 체크리스트가 내장된 AI 에이전트입니다. 팀원이 매번 프롬프트를 작성하거나 디자인 가이드를 참고할 필요가 없습니다.
스킬 목록

등록된 4개 스킬

각 스킬은 독립적으로 작동하며, 조합해서 사용할 수 있습니다.

🎯
Cochl Pitch Deck
/cochl-pitch-deck
최소한의 정보(제품명, 핵심 메시지, 데이터 포인트)만 주면 Cochl 브랜드 스타일의 프레젠테이션을 처음부터 자동으로 생성합니다. 13슬라이드 구조를 자동 매핑하고 HTML + PPTX 두 형식으로 동시 출력합니다.
HTML 출력 PPTX 출력 텍스트/JSON 입력
🎨
Pitch Deck Skill (슬라이드 수정)
/pitch-deck-skill
기존 덱의 특정 슬라이드 내용만 교체하거나, 다른 제품용 새 덱을 같은 Cochl 비주얼 시스템으로 제작합니다. 30개 레이아웃 아카이브 + Cochl PT 템플릿 정확한 치수가 내장됩니다.
HTML 출력 PPTX 출력 기존 덱 수정
💻
Frontend Design
/frontend-design
React/Vite 프로젝트, 웹 컴포넌트, HTML 페이지에 Cochl 디자인 시스템을 적용합니다. 프로젝트 유형을 자동 감지해 React DS 모드 / Pitch Deck 모드 / HTML 모드 중 적절한 방식으로 동작합니다.
React 지원 HTML 출력 컴포넌트 입력

React 프로젝트에 @cochlearai/ui, @cochlearai/util을 자동 설치하고 BaseTheme으로 루트를 래핑합니다. .npmrc 설정, 환경변수 토큰, 빌드 검증까지 전 과정을 자동화합니다.
React 전용 package.json 필요

Installation

팀원 설치 방법

스킬 파일은 레포지토리의 .claude/skills/ 폴더에 있습니다. 처음 한 번만 설치하면 됩니다.
사전 조건: Claude Code CLI가 설치되어 있어야 합니다. 설치되지 않은 경우 npm install -g @anthropic-ai/claude-code 를 먼저 실행하세요.
  1. 1
    레포지토리 클론
    프로젝트 레포를 클론하거나 최신 상태로 pull합니다.
    Terminal
    git clone git@github.com:cochlear-ai/[레포명].git
# 또는 기존 클론이 있다면
git pull origin main
  2. 2
    설치 스크립트 실행
    프로젝트 루트에서 아래 명령어를 실행합니다. 스킬 파일들이 ~/.claude/skills/에 복사됩니다.
    Terminal — 프로젝트 루트에서 실행
    bash .claude/install-skills.sh
    설치 성공 시 아래와 같이 출력됩니다:
    ✓ cochl-pitch-deck.md
✓ pitch-deck-skill.md
✓ frontend-design.md
✓ install-cochl-design-system/ (directory)

Done! Restart Claude Code, then try:
  /cochl-pitch-deck
  3. 3
    Claude Code 재시작
    설치 후 Claude Code를 완전히 종료하고 다시 실행해야 새 스킬이 인식됩니다.
    IDE 확장(VS Code, JetBrains)을 사용하는 경우 확장도 재시작합니다.
  4. 4
    설치 확인
    Claude Code에서 /cochl-pitch-deck을 입력했을 때 스킬이 로드되면 설치 완료입니다.
    Claude Code 프롬프트
    /cochl-pitch-deck  ← 이걸 입력해서 스킬이 뜨면 성공
스킬 업데이트: 스킬 파일이 업데이트되면 git pull 후 설치 스크립트를 다시 실행하세요. Claude Code를 재시작하면 새 버전이 적용됩니다.

Skill 01

/cochl-pitch-deck

처음부터 새 덱을 만들 때 사용합니다. 최소 입력으로 최대 결과물을 냅니다.
무엇을 줘야 하나요?
아래 중 하나 이상이면 충분합니다:
  • 제품/회사명
  • 발표 대상 (투자자, 정부, 기업 구매자)
  • 핵심 메시지, 데이터, 불릿 리스트
  • JSON 파일, PRD, 텍스트 브리프
  • 기존 제안서나 문서 파일 경로
무엇을 만들어주나요?
  • 13슬라이드 자동 구조 매핑
  • break 슬라이드 3개 자동 삽입
  • HTML (브라우저에서 바로 보기)
  • PPTX (PowerPoint에서 편집)
  • 키보드 네비게이션 포함

Quality Assurance

덱 생성 후 QA 체크리스트

스킬이 자동으로 검증하지만, 출력물을 사람이 최종 확인할 때 사용하세요.
시각·구조 체크
  • 커버/CTA 슬라이드에 PT 배경 이미지 적용됨
  • 일반 콘텐츠 슬라이드는 흰 배경 + 검정 텍스트
  • 각 h2에 grad-text span이 정확히 1개
  • 카드 본문이 2문장 이하
  • 하단 페이지 네비게이터 정상 작동
  • 커버 좌하단에 Cochl 로고 존재
파일·기술 체크
  • HTML 파일이 완전 self-contained (외부 의존 없음)
  • PPTX 파일이 정상 생성됨 (16슬라이드)
  • 키보드 화살표 키 네비게이션 작동
  • IBM Plex Sans 웹폰트 로드됨
  • PPTX 슬라이드 크기 10" × 5.625"
  • 하드코딩된 컬러 없음 (CSS 변수 사용)
FAQ

자주 묻는 질문

Q. 스킬을 입력해도 아무 반응이 없어요.
설치 스크립트를 실행했는지 확인하세요 (bash .claude/install-skills.sh). 설치 후 반드시 Claude Code를 완전히 재시작해야 합니다. 그래도 안 된다면 ls ~/.claude/skills/로 파일이 있는지 확인하세요.
Q. PPTX 파일이 생성되지 않아요.
python3 -c "import pptx; print(pptx.__version__)"로 python-pptx가 설치되어 있는지 확인하세요. 없다면 pip3 install python-pptx를 실행합니다.
Q. 스킬 파일이 업데이트됐는데 어떻게 적용하나요?
git pull 후 bash .claude/install-skills.sh를 다시 실행하고 Claude Code를 재시작합니다.
Q. /install-cochl-ds 를 쳤는데 401 에러가 나요.
COCHLEARAI_NPM_TOKEN 환경변수가 설정되지 않았습니다. GitHub Packages 읽기 권한 토큰을 발급 받아 export COCHLEARAI_NPM_TOKEN=ghp_xxx로 설정하세요.
Q. 새 스킬을 추가하려면 어떻게 하나요?
.claude/skills/에 my-skill.md 파일을 추가하고 상단에 name: my-skill frontmatter를 작성하세요. 설치 스크립트가 자동으로 포함합니다.
