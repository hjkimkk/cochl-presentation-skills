Project Overview

Scope

  • Discovery and Research I'll audit the existing web dashboard, interview manager-level users to map their monitoring workflows, and define the data and functional requirements for the mobile experience. A user journey map and functional requirements document will frame all subsequent design decisions;
  • UX Design I will design the information architecture and interaction flows for the mobile dashboard. Low-fidelity wireframes covering all primary screens — project monitoring, detected data, notifications, and data sharing — will be presented for alignment before any visual design begins;
  • UI Design After the UX direction is approved, I will design all high-fidelity screens in Figma. This includes default states, empty states, notification views, and the data sharing flow. All screens will be optimized for mobile-first legibility and one-handed use;
  • Handoff Final designs, annotated interaction specs, and component documentation will be delivered. I'll conduct a handoff walkthrough with the engineering team and update the design system with any new mobile components.

Milestones

Milestone 1: Discovery and Research, UX Design

  • Deliverables:
    • Manager user interviews — 3–5 sessions, synthesized into key insight themes
    • Functional requirements document — data views and actions for mobile vs. web dashboard:
      • Monitor: Project info, SDK Device status, Detected data, Usage, Insights
      • Act: Copy project key / session UUID, share data directly with team members
      • Receive: Push notifications for custom alerts and major events
      • Extend: SDK device location tagging (floor, building, placement)
    • User journey map — 5-stage flow (Project create → Access → Monitor → Detect & Share → Respond)
    • Information architecture for all mobile screens
    • Low-fidelity wireframes for all primary screens (mobile-optimized)
  • UX Alignment Session
    A structured review of the wireframes will be held with the PM and engineering lead to confirm scope, prioritize features for the initial release, and align on the data sharing model before high-fidelity design begins;

Milestone 2: UI Design, Handoff

  • Deliverables 1:
    • High-fidelity Figma screens — all states (default, loading, empty, error):
      • Home / summary view with key metrics
      • Project list + project detail
      • SDK device monitoring (Essentials + location)
      • Detected data and analytics / insights view
      • Custom notification settings
      • Data sharing flow (share directly to team members)
    • Design system update — new mobile components documented
  • Deliverables 2:
    • Annotated interaction specs for engineering
    • Handoff walkthrough session with engineering team
    • Final Figma file with organized layers and component library
